# 🗑️ TIW: Today I Wasted (今天我又浪费了什么)

🇺🇸 [English](README.md) | 🇰🇷 [한국어](README-kr.md) | 🇨🇳 [中文](README-cn.md)

> *"明明5分钟就能搞定，为什么非要花4个小时去自动化呢？"*

欢迎来到我生产力的数字坟场。当那些“卷王”们正在用 **TIL (Today I Learned)** 仓库填满他们的脑沟回时，我却在这里记录下我最后几个脑细胞蒸发的瞬间。

如果你在寻找高效的代码——不管你是谁——现在离开还来得及。如果你想体验一下精神冲击，那欢迎入坑。

---

## 📂 废料篮 (这里面有什么)
这是一个杂乱无章的归档。如果我把有限的生命浪费在屏幕前搞定了什么，那它就是浪费时间，且必须放在这里。

### 🧮 1. 数学与算法挣扎 (`/algorithms`)
*   那些可能违反了人类已知所有“整洁代码”准则的解题方案。
*   写出可怕的 $O(N^{N^N})$ 暴力循环，并向我的 CPU 道歉。
*   因为大脑已经忘记怎么做基础算术，所以只能用代码来解决数学问题。

### 🐛 2. Bug 墓地 (`/bug-fixes`)
*   记录那些昨天运行完美，今天却莫名其妙崩溃的代码奇迹。
*   花了5个小时翻遍 GitHub Issues，结果发现修复方法只是漏了一个逗号。
*   在修复了自己亲手制造的 Bug 后，还要假装自己是个天才。

### 📚 3. 毫无羞耻感的自我教育 (`/til`)
*   那些别人5年前就搞明白的“标准” TIL 笔记。
*   在无聊得昏昏欲睡之前，强行阅读官方文档后留下的乱糟糟总结。
*   学了2个小时就彻底放弃的各种随机技术栈。

## 📜 仓库指南

1. **零组织结构：** 如果创建文件夹太麻烦，文件就会直接扔在根目录下。
2. **拒绝润色：** 没有精美的排版或美化。你看到的只是一个绝望开发者的原始、未经编辑的焦虑。
3. **保持不一致：** 随心所欲地提交代码。真正浪费时间的人从不遵循计划。
